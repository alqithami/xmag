\begin{abstract}
5G-enabled Internet of Things (IoT) environments require intrusion detection systems that are lightweight, explainable, and robust to unknown attacks. Existing IDS pipelines often assume centralized feature access and use explainability only as a post-hoc reporting tool. This paper proposes \framework{}, an explanation-aware multi-agent IDS in which lightweight local agents transmit compact evidence messages instead of raw traffic. Each 24-byte agent message contains top-1 class evidence, top-1 explanation evidence, and a local anomaly scalar. A coalition classifier predicts known attacks from these compact messages, while a separate composite open-set head combines message residuals, owner-agent uncertainty, and anomaly evidence to detect unknown attack families. On real 5G-NIDD leave-one-attack-family-out experiments, the current single-seed \cosmethod{} result achieves mean known macro-F1 of 0.9979, mean unknown AUROC of 0.9502, and mean unknown recall of 0.8517 at approximately 5\% false-alarm rate, while retaining a 24-byte per-flow message. The results also show that relying on coordinator uncertainty alone fails on UDPFlood; composite open-set scoring improves the worst-case unknown recall from 0.0133 to 0.4250 without increasing message size. This draft freezes the main method, reports the first real 5G-NIDD result tables, and defines the next seed-stability sweep required before final manuscript claims.
\end{abstract}

\noindent\textbf{Keywords---} Intrusion detection; 5G security; IoT security; multi-agent systems; explainable AI; open-set recognition; zero-day detection; lightweight machine learning; anomaly detection.

\section{Positioning}

This manuscript is positioned around a central idea: explanations should not only justify intrusion detection decisions after they are made; in distributed IoT/5G environments, compact explanation-aware messages can become the communication primitive through which lightweight security agents collaborate. The current method, \framework{}, moves beyond explanation-only communication by adding a dedicated composite open-set head. The paper's key claim is therefore not that explanations alone solve intrusion detection. The stronger claim is that compact explanation-aware communication is viable for known-attack classification, while robust zero-day behavior requires a separate open-set scoring mechanism.

\section{Motivation and Research Gap}

IoT devices connected through 5G networks create a distributed, heterogeneous, and resource-constrained attack surface. Traditional centralized IDS models often assume full access to all traffic features, while many explainable IDS studies apply SHAP, LIME, or feature-importance explanations only after training. Such designs do not directly address the communication and computation constraints of distributed IoT/edge security agents.

The research gap addressed here is the lack of an IDS formulation that simultaneously handles: (i) multi-agent cooperation, (ii) strict communication budgets, (iii) interpretable evidence exchange, and (iv) leave-one-attack-family-out unknown-attack evaluation. The initial explanation-only approach preserved known-class performance but exposed failure cases for open-set detection. This motivates \framework{}, where the classification head and open-set head are separated.

\section{Method: \framework{}}

\subsection{Agent Message}

Let $a_i \in \mathcal{A}$ denote a lightweight IDS agent. Each agent observes a local flow vector $x_i$ and produces a local class-probability vector $p_i$, an explanation contribution vector $\phi_i$, and an anomaly score $z_i$. Instead of transmitting raw features, the agent sends a compact message:

\begin{equation}
    m_i = \left[\operatorname{TopM}(p_i, m),\; \operatorname{TopK}(\phi_i, k),\; z_i\right].
\end{equation}

The current main method freezes $m=1$ and $k=1$. With the byte accounting used in the implementation, this gives a 24-byte per-flow message:

\begin{equation}
    \cosmethod{} = \operatorname{Top1ClassEvidence} + \operatorname{Top1ExplanationEvidence} + \operatorname{LocalAnomalyScalar}.
\end{equation}

\subsection{Known-Attack Classification Head}

The classification head is a coalition-level coordinator trained on the compact messages $m_i$. It predicts the known-class label $\hat{y}$:

\begin{equation}
    \hat{y} = g_{\text{cls}}(m_i).
\end{equation}

In the current implementation, the local agents use lightweight random-forest classifiers and the coordinator is a logistic one-vs-rest classifier over compact message features. This is intentionally simple to keep the first result reproducible and lightweight.

\subsection{Composite Open-Set Head}

The open-set head does not rely only on the coordinator's maximum class probability. Earlier experiments showed that coordinator uncertainty failed badly for UDPFlood. Instead, \framework{} computes a composite score from complementary signals:

\begin{equation}
    s_{\text{open}} = \gamma \max(s_{\text{resid-owner}}, s_{\text{uncert-anom}}) + (1-\gamma) s_{\text{uncert-anom}},
\end{equation}

where $s_{\text{resid-owner}}$ combines message-prototype residual and owner-agent uncertainty, and $s_{\text{uncert-anom}}$ combines owner-agent uncertainty and local anomaly evidence. The frozen score used by \cosmethod{} is:

\begin{equation}
\texttt{linear\_max\_resid\_owner\_b025\_\_uncert\_anomaly\_b05\_gamma\_0.25}.
\end{equation}

This design separates the role of the coordinator: classification is learned from compact messages, while unknown detection is driven by a composite direct score that remains sensitive to uncertainty, anomaly shift, and message residuals.

\section{Experimental Protocol}

The current experiments use the real 5G-NIDD encoded CSV file with leave-one-attack-family-out evaluation. Each attack family is held out as unknown, while the remaining known attack families and benign traffic are used for known-class training and testing. The held-out families are HTTPFlood, ICMPFlood, SlowrateDoS, SYNFlood, SYNScan, TCPConnectScan, UDPFlood, and UDPScan.

The reported metrics are known macro-F1, unknown AUROC, unknown recall at a validation-calibrated threshold, known false-alarm rate at that threshold, message bytes per flow, and runtime. The current result uses a single random state, so the next experiment is a stability sweep over random states $\{7,42,123\}$.

\section{Interpretation}

The main finding is that compact explanation-aware communication is viable, but open-set detection should not be delegated to the coordinator confidence alone. The UDPFlood failure case is especially important: with the same 24-byte message, \dhmethod{} produced unknown AUROC of 0.4893 and recall of 0.0133, while \cosmethod{} improved these to 0.7899 and 0.4250. This supports the design choice of a composite open-set head.

The current result should be stated carefully. \framework{} does not solve open-set intrusion detection completely; rather, it materially improves worst-case zero-day behavior under a strict communication budget while preserving known-class detection.

\section{Expected Contributions}

The paper is now organized around four contributions:

\begin{enumerate}[leftmargin=*]
    \item A compact multi-agent IDS message format that combines top-1 class evidence, top-1 explanation evidence, and local anomaly evidence in 24 bytes per flow.
    \item A dual-head architecture separating known-attack classification from unknown-attack scoring.
    \item A composite open-set score that improves worst-case unknown recall compared with coordinator uncertainty alone.
    \item A real 5G-NIDD leave-one-attack-family-out evaluation protocol with communication-cost accounting.
\end{enumerate}

\section{Limitations}

The current result is single-seed and uses transparent hash-based agents rather than a real deployment mapping. The local explanation token currently uses lightweight feature-importance contribution proxies rather than full SHAP/LIME. These are acceptable for the first empirical draft but must be explicitly reported. Future versions should add seed stability, noisy-agent robustness, physical agent mappings where available, and SHAP/LIME comparison.

\section{Conclusion}

This draft freezes \framework{} as the main method. The result supports a precise and defensible paper claim: compact explanation-aware multi-agent communication can preserve known-attack classification and substantially improve worst-case zero-day recall when paired with a composite open-set head. The next step is to run the seed-stability sweep and then integrate the final statistics into the manuscript.
